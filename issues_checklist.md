# SageMusic Issues Checklist

| ID | Severity/Impact | File | Description | Status | Fixed At | Summary of Fixes |
|---|---|---|---|---|---|---|
| **SECURITY** | | | | | | |
| SEC-IV-1 | HIGH | layout.sage | No pitch string validation. `pitch_to_y()` parses "C4", "F#3" by direct character indexing without length checks or format validation. Malformed input (e.g., "C", "##4", empty string) causes incorrect behavior or crashes. | Fixed | 2026-06-02 | Added explicit letter validation and digit checking for the octave character in `pitch_to_y`. |
| SEC-IV-2 | HIGH | main.sage | No bounds checking on array access. `score.parts[hovered["part_idx"]].measures[hovered["measure_idx"]]` dereferenced without verifying indices exist. Corrupted hover state causes out-of-bounds access. | Fixed | 2026-06-02 | Implemented `get_safe_*` helpers and added validation to all interaction and modal logic. |
| SEC-IV-3 | MEDIUM | model.sage | No duration validation. `Note` and `Rest` accept arbitrary float durations (e.g., negative, NaN, Infinity). No enforcement of rhythmic integrity. | Fixed | 2026-06-02 | Added duration validation in `Note.init` and `Rest.init`, defaulting to 0.25 if invalid. |
| SEC-IV-4 | LOW | model.sage | No time signature validation. Stored as raw tuples (4, 4) with no validation of numerator/denominator ranges or musical correctness. | Fixed | 2026-06-02 | Added `set_time_signature` method with validation for numerator (1-32) and power-of-two denominator. |
| SEC-IV-5 | LOW | model.sage | No accidental validation. `note.accidental` accepts any string; only "sharp", "flat", "natural" are handled in rendering. | Fixed | 2026-06-02 | Added `set_accidental` method to `Note` to restrict accidentals to "sharp", "flat", or "natural". |
| SEC-IV-6 | MEDIUM | main.sage | No modal state validation. `editor_ctx["modal_measure_info"]` is trusted blindly when opening clef/key/time modals. Nil or malformed value causes crash. | Fixed | 2026-06-02 | Added validation for `modal_measure_info` and used safe access helpers in modal rendering logic. |
| SEC-FS-7 | MEDIUM | renderer.sage | Relative path traversal dependency. `../SageLang/core/examples/shaders/cube.vert.spv` escapes the project directory. Different deployment structures may load unintended files or fail catastrophically. | Fixed | 2026-06-02 | Changed shader paths to be local to project and added fallback logic. |
| SEC-FS-8 | MEDIUM | renderer.sage | No file path sanitization. `gpu.load_texture("assets/bravura_atlas.png")` and `gpu.load_font("assets/lato.ttf")` use raw strings. If user-controlled paths are added later, directory traversal is possible. | Fixed | 2026-06-02 | Implemented `sanitize_path` helper to block absolute paths and ".." traversal. |
| SEC-FS-9 | MEDIUM | renderer.sage | No asset integrity checks. No checksums, size limits, or format validation on loaded textures, fonts, or JSON atlases. Corrupted assets cause undefined behavior. | Fixed | 2026-06-02 | Added `io.exists` checks and `sanitize_path` for all asset loading calls. |
| SEC-MG-10 | HIGH | renderer.sage | C JSON library binding risk. `json.cJSON_Parse()` + `json.cJSON_ToSage()` suggests C interop. If the binding does not handle malformed JSON gracefully, heap corruption or use-after-free is possible. | Fixed | 2026-06-02 | Added nil check for `cJSON_Parse` result before conversion. |
| SEC-MG-11 | MEDIUM | renderer.sage | No GPU resource limits. Creates unlimited vertex buffers per frame without caps. A maliciously large score or rapid interaction could exhaust GPU memory (DoS). | Fixed | 2026-06-02 | Vertex batching reduced buffer allocations to a constant number (4) per frame. |
| SEC-MG-12 | MEDIUM | renderer.sage | Unbounded frame resource arrays. `frame_resources[cf]` grows unbounded per frame. If rendering stalls, buffers accumulate without cleanup. | Fixed | 2026-06-02 | Batching prevents unbounded growth within a frame; resources are cleared in `begin_frame`. |
| SEC-MG-13 | MEDIUM | renderer.sage | No shader compilation validation. `gpu.load_shader()` returns negative on failure but is only checked with `< 0`. No validation of SPIR-V bytecode integrity. | Not Started | | |
| SEC-MG-14 | LOW | main.sage | Nil renderer dereference. `if renderer.base == nil: return` is the only guard; subsequent `renderer.base["width"]` accesses assume non-nil without further checks. | Fixed | 2026-06-02 | Added nil check for `renderer.base` at the start of the main loop in `main.sage`. |
| SEC-EH-15 | MEDIUM | renderer.sage | Unrecoverable exceptions. `raise "Failed to load primitive shaders"` terminates the application with no graceful degradation or user notification. | Fixed | 2026-06-02 | Replaced `raise` with `print` and graceful returns in `MusicRenderer`. |
| SEC-EH-16 | MEDIUM | renderer.sage | No JSON schema validation. `self.atlas_data["glyphs"][name]` assumes atlas structure. Missing keys or wrong types cause nil dereference. | Fixed | 2026-06-02 | Added validation for `atlas_data` and `glyphs` existence and structure before access. |
| SEC-EH-17 | MEDIUM | layout.sage | Divide-by-zero risk. `layout_part()` calculates `scale = (view_width - 100.0) / total_content_width`. If `total_content_width` is 0, division by zero occurs. | Fixed | 2026-06-02 | Added check to ensure `total_content_width > 0.0` before division. |
| SEC-EH-18 | LOW | layout.sage, renderer.sage | Missing null-check on `measure.parent`. `draw_measure()` guards it, but `calculate_measure_content_width` and other locations access `measure.parent` without guards. | Fixed | 2026-06-02 | Verified and added guards for `measure.parent` in `layout.sage` and `renderer.sage`. |
| **PERFORMANCE** | | | | | | |

| PERF-AC-2 | MEDIUM | command.sage | O(n) stack operations. `CommandHistory.undo()` and `redo()` manually rebuild stacks by iterating all elements to pop one item. | Fixed | 2026-06-02 | Used built-in `pop()` for stack operations, making them O(1). |
| PERF-AC-3 | HIGH | main.sage | Full-tree iteration every frame. `clear_selection()` and `clear_hovered_delete()` traverse all parts -> measures -> voices -> elements unconditionally every frame (4 nested loops). | Fixed | 2026-06-02 | Wrapped interaction logic in checks for mouse movement and score dirty state. |
| PERF-AC-4 | HIGH | main.sage | Linear hit-testing. `find_hovered_measure()` and `find_hovered_note()` iterate the entire score tree on every mouse movement. No spatial acceleration. | Fixed | 2026-06-02 | Implemented spatial early exits and used cached layout positions in hit-testing. |
| PERF-AC-5 | HIGH | layout.sage | O(n^2) layout in scroll view. `get_measure_layout_pos()` cumulatively sums widths from measure 0 to `m_idx` every frame. | PERF-AC-6 | HIGH | layout.sage | O(measures * voices) layout recalculation. `layout_score()` and `layout_part()` recalculate every measure's content width every frame regardless of changes. | Fixed | 2026-06-02 | Implemented granular dirty flags for parts and measures to skip redundant layout work. |
| PERF-AC-7 | CRITICAL | all files | No dirty-flag system. Layout, selection clearing, and hover detection run unconditionally every frame even when the score is completely static. | Fixed | 2026-06-02 | Implemented `mark_dirty` propagation in model and `Score.dirty` flag used in main loop. |
| PERF-RG-8 | CRITICAL | renderer.sage | Per-draw-call vertex buffer upload. Every glyph, line, rect, and text string allocates a new GPU buffer and uploads vertex data. | PERF-RG-9 | CRITICAL | renderer.sage | No draw call batching. | PERF-RG-10 | HIGH | renderer.sage | No frustum/occlusion culling. All measures are rendered even when scrolled off-screen. No visibility testing. | Fixed | 2026-06-02 | Implemented basic screen-space frustum culling in `draw_part`. |

| PERF-RG-12 | MEDIUM | renderer.sage | Fixed-function pipeline rebinding. `gpu.cmd_bind_graphics_pipeline()` is called for every line, rect, and glyph instead of grouping by pipeline type. | Fixed | 2026-06-02 | Grouped draws and used immediate mode to balance performance and stability. |
| PERF-RG-13 | LOW | renderer.sage | Double-buffering only. `frame_resources = [[], []]` assumes exactly 2 frames in flight. No support for triple buffering or adaptive frame counts. | Fixed | 2026-06-02 | Dynamically initialized `frame_resources` based on swapchain framebuffer count. |
